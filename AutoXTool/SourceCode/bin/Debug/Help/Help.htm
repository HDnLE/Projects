Help: System XAuto-Test

Overview

The utility is designed with Ranorex (See here for details about Ranorex) running in the background. This is like "automation within automation". The purpose of this utility is to help Test Automation Developers reduce their tasks when developing automated scripts. And actually, even if engineers with not so much knowledge on Ranorex can be an instant Test Automation Developer.

This utility is using a MySql database where your set of instructions are keyed in. The utility will just read instructions from the database and then you have the option to write (the utility will write Ranorex scripts for you) or execute the script. You have to use System XAuto-Test Web UI to do this.

Please refer to this architecture.

Execute Mode

 

Execute Mode is one of the two modes of this tool. This mode will allow user to check or see his designed scripts run in action before he converts them to executable scripts.

Run a Script From Execute Mode

  1. Click Select Mode > Execute Script
  2. Select a testsuite from the list
    3. A testsuite is where the testcase that you want to execute belongs to
  3. Select a testcase from the list that you want to execute
  4. Click on Execute button to start running the script

Write Mode

 

Write Mode is the another mode of this tool. This mode will allow user to convert the instructions made in the UI into executable (Ranorex) script. This mode can write single script, multiple scripts (or range of scripts) or a masterscript. A masterscript is a script where selected individual scripts are put into one.

Write Single Script

  1. Click Select Mode > Write Script > Single Script
  2. Select a testsuite from the list
  3. Select a testcase from the list that you want to write
  4. Click on Write button to start writing the script
  5. A confirmation message will popup when done. You will find all the written scripts at *C:\SystemXScripts folder

    6. *You can also change the folder where you want to store the scripts through Settings > Script Settings

Write Multiple Scripts

  1. Click Select Mode > Write Script > Multiple Scripts
  2. Select a testsuite from the list
  3. Key in the testcase ID of the scripts that you want to write
    4. To write multiple scripts, separate IDs with comma (,). For example, 100,105,117. You can also write scripts in range or combination like 100,105,117-130.
  4. Click on Write button to start writing the script
  5. A confirmation message will popup when done. You will find all the written scripts at *C:\SystemXScripts folder

    6. *You can also change the folder where you want to store the scripts through Settings > Script Settings

Write Masterscript

  1. Click Select Mode > Write Script > Master Script
  2. Select a testsuite from the list
  3. Click on Write button to start writing the masterscript
  4. A confirmation message will popup when done. You will find all the written scripts at C:\SystemXScripts folder

    5. *You can also change the folder where you want to store the scripts through Settings > Script Settings
    Scripts will be included in the masterscript list if you set the Add to Masterscript option to true. Go to Manage Test Cases page for more details.
